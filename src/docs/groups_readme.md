# 操作說明：群組與物件設定

本系統旨在協助使用者進行空間規劃與物件管理。  
基本操作流程為 **先設定「群組」(Group)**，再於群組內設定 **「物件」(Object)**。

---

## 1. 群組設定 (Group Setup)

在進行任何物件配置之前，您需要先建立或選擇一個群組。  
群組可以被視為「邏輯上的容器」，用於組織相關的物件，例如：

- **建立新群組**
  - 定義群組名稱  
  - （可選）設定容量限制或其他相關屬性  
- **選擇現有群組**
  - 從群組列表中挑選一個已存在的群組  
  - 在該群組內進行物件新增、編輯或刪除動作  

>  群組的設定通常會影響其內部物件的行為或限制，例如最大可放置數量、可使用的空間範圍等。

---

## 2. 物件設定 (Object Setup)

一旦群組設定完成，您便可以在選定的群組內新增或管理物件。  
物件代表您希望放置或操作的實體（例如各種箱子、零件、貨物等）。

### 2.1 新增物件 (Create Object)

新增物件時，通常需要設定以下資訊：

- **物件類型 (Object Type)**  
  例如：箱子、圓柱體、特殊形狀…等。
- **基本屬性 (Basic Attributes)**  
  為物件指定：
  - 唯一的識別碼（ID）
  - 物件名稱或說明文字

### 2.2 物件狀態 (Object State)

每個物件都可以設定其「當前狀態」，這有助於系統追蹤物件的生命週期或特定屬性。  
常見狀態包括（實際名稱可依系統 UI 調整）：

- **待放置 (Pending)**：  
  物件已建立，但尚未在空間中決定具體位置。
- **已放置 (Placed)**：  
  物件已成功放置在群組內某個位置。
- **溢出 (Overflow)**：  
  因空間不足或其他限制，物件目前無法放入群組中。
- **損壞 (Damaged)**：  
  代表物件本身的物理狀態，有可能不再可用。
- **鎖定 (Locked)**：  
  防止物件被移動、刪除或修改。
- **其他自訂狀態 (Custom States)**：  
  可依實際業務需求新增，例如「待檢查」、「待補貨」等。

### 2.3 物件尺寸 (Object Dimensions)

物件的尺寸是其在三維空間中的實際大小，對空間規劃與碰撞檢測非常重要。  
系統通常需要您為每個物件輸入：

- **長度 (Length)**
- **寬度 (Width)**
- **高度 (Height)**

這些尺寸將用於：

- 計算物件所佔用的空間體積  
- 檢查物件之間是否重疊或超出容器範圍  
- 協助演算法進行空間利用率與擺放位置的最佳化

---

>  建議使用流程：
> 1. 先建立或選擇一個「群組」  
> 2. 在該群組底下新增多個「物件」  
> 3. 為物件設定尺寸與狀態  
> 4. 交由系統進行 3D 打包與空間配置
